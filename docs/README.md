## 📝 文档规范

### 文件命名
- 使用英文命名，单词间用下划线或连字符分隔
- 重要文档使用大写（如 README.md, CHANGELOG.md）
- 指南类文档建议使用描述性名称（如 QUICKSTART.md, LOCAL_DEVELOPMENT.md）

### 文档结构
- 使用 Markdown 格式
- 包含清晰的标题层级
- 提供目录索引（对于长文档）
- 包含代码示例和命令说明
- 添加相关文档的链接

### 文档位置
- **所有文档统一放在 `docs/` 目录下**
- 项目根目录只保留 `README.md`
- 文档间使用相对路径引用
